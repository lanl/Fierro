Command Line Interface Guide

Entry Points

The Fierro command line interface offers a single interface into all of the Fierro backend tools. The current options available on your machine can be explored by running 

fierro -h

This will list as subcommands all of the available backend applications.

Parallel Explicit Solver

The parallel explicit solver can be invoked through 

fierro parallel-explicit

As an argument, it expects a Yaml configuration file with the following configuration options specified:

Simulation Parameters
solver_type Type of solver used {SGH}
num_dims Dimensionality of Problem {2, 3}
fea_modules List of FEA Modules to use in the optimization
FEA Module
fea_module_type Type of moduel {Elasticity, Heat_Conduction, SGH, Inertial, Thermo_Elasticity, Eulerian}
boundary_conditions List of FEA Boundary Conditions
loading_conditions List of FEA Loading Conditions
FEA Boundary Condition
id Unique condition id
surface Type of surface {x_plane, y_plane, z_plane, cylinder, sphere, readFile}
condition_type The type of constraint {fixed_displacement, fixed_temperature}
value The value to enforce along the boundary
displacement_value The displacement to enforce along the boundary
temperature_value The temperature to enforce along the boundary
plane_position The position of the plane along its axis
FEA Loading Condition
id Unique condition id
surface Type of surface {x_plane, y_plane, z_plane, cylinder, sphere, readFile}
condition_type The type of constraint {surface_traction, surface_heat_flux}
flux_value The flux to enforce along the boundary
specification The directionality of the traction {normal, coordinated}
component_x The x component of the loading force
component_y The y component of the loading force
component_z The z component of the loading force
plane_position The position of the plane along its axis
Input Options
mesh_file_name Filepath to the mesh to load
mesh_file_format Format of the mesh file {ensight, tecplot, vtk, ansys_dat}
element_type The type of elements contained in the mesh file {quad4, quad8, quad12, hex8, hex20, hex32}
zero_index_base
Output Options
graphics_step_frequency Output current mesh every "graphics_step_frequency" optimization steps.
graphics_step
output_format Mesh output format {vtk}
